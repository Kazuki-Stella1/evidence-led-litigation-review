目安：2分30秒から2分45秒。英語字幕又は `demo-script-en.md` の英訳を提出資料に含める。

## 0:00–0:17　問題

画面：作品名、`sample/workflow/input/manifest.json`

「法的書面を何度も直す過程では、PDF、スクリーンショット、写真、文書を何度も開き直します。その間に、号証がずれ、事実と推論が混ざり、因果関係が抜け、相手方証拠の確認にもばらつきが生じます。」

## 0:17–0:39　二層構造と二段階確認

画面：アーキテクチャ図、README、スキル

「Evidence-Led Litigation Reviewは二層を分けます。決定的なPythonツールは、ローカルで証拠台帳、マップ、不整合候補、位置付き校正一覧を作ります。Codex上のGPT-5.6は、直接立証と推論を分け、因果関係、損害、最強反論、表現を検討します。普段はテキストで速く確認し、重要事項は原本へ戻ります。」

## 0:39–1:02　証拠ワークフローを実行

画面：ターミナル

```bash
python scripts/run_workflow_demo.py
```

「完全に架空の双方当事者による紛争を使います。一つのコマンドで、七つの証拠・書面台帳、右上がりの時系列図、甲乙双方の論理検証と校正一覧を生成します。PythonはモデルAPIを呼びません。任意のローカルツールがあれば、PDFと画像OCRにも対応します。」

## 1:02–1:22　マップと相手方検証

画面：`evidence-map.html`、`logic-review.md`

「右側には号証、日時、頁、確認状態、左側には事実、立証趣旨、因果関係、矛盾点、推論、法的評価があります。双方を絞り込めます。架空例では、存在しない乙3と、相手方の直接主張が推論に依存する点を検出します。これは法的結論ではなく確認候補です。」

## 1:22–1:44　位置付き校正を確認

画面：ターミナル、`sample/proofreading/output/proofreading/proofreading-review.md`

```bash
python scripts/run_proofreading_demo.py
```

「校正デモは、書面、抽出テキストの行、文字位置、修正候補、理由、確度を付けて七件を一覧化します。重複入力、三つの誤変換、括弧欠落、未登録号証、明示的な甲1から甲2への修正を検出し、Markdown、JSON、表計算用CSVへ出力します。」

## 1:44–1:59　従来のbefore／after

画面：ターミナル、安定版レポート

```bash
python scripts/run_demo.py
```

「従来デモも、十一件の構造警告がある37点から、決定的警告ゼロの100点へ移ります。これは勝率ではありません。この規則で書面上の警告が残るかを示すだけで、法的・証拠上の確認は残ります。」

## 1:59–2:27　CodexとGPT-5.6

画面：スキル、README、テスト、ビルドログ

「Codexは、プラグイン構造、抽出・校正実装、文書処理、画像OCRテスト、架空fixture、マップ、プライバシー検査、文書化、修復、リリース検証を加速しました。GPT-5.6は、直接立証と推論、時系列から損害までの連鎖、別原因と医療事象、損害期間、最強反論、証拠に忠実な表現、匿名化だけでは再識別を防げないという判断を担いました。」

## 2:27–2:45　締め

画面：全テスト、構成、作品名

「抽出テキストを原本の代わりにせず、日常確認を速くします。We did not build a machine that decides who wins. We built a workflow that makes every important claim easier to verify, challenge, and improve.」
